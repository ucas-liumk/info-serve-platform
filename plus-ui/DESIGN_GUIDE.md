# 信息中心数智服务平台基础设计规范 v0.1

本文档先服务于门户首页、工具即用、需求反馈等门户侧界面。后台管理后续再单独统一。

## 设计定位

- 门户侧面向内部用户，风格应清爽、可信、易扫描。
- 优先保证功能效率，不做强营销感页面。
- 视觉基调参考门户首页和工具即用：浅蓝白底、深蓝文字、克制边框、轻阴影。

## 色彩

| 用途       | 色值      | 说明                     |
| ---------- | --------- | ------------------------ |
| 主色       | `#1260e8` | 主按钮、选中态、重点图标 |
| 主色浅底   | `#edf4ff` | 选中菜单、标签、浅色块   |
| 标题文字   | `#0b1833` | 页面标题、卡片标题       |
| 正文文字   | `#25395f` | 说明文字、表单内容       |
| 次级文字   | `#53668f` | 提示、时间、辅助信息     |
| 弱提示文字 | `#8a97af` | 空状态、时间弱化         |
| 页面底色   | `#f7faff` | 门户应用页背景           |
| 卡片底色   | `#ffffff` | 卡片、弹窗、输入容器     |
| 边框       | `#e1e9f6` | 卡片、弹窗、列表边框     |
| 错误/删除  | `#d93026` | 删除按钮、危险操作       |

## 字体与层级

- 字体族：`HarmonyOS Sans SC`, `PingFang SC`, `Microsoft YaHei`, `sans-serif`。
- 页面一级标题：`30px - 34px`，字重 `800+`。
- 弹窗标题：`22px - 24px`，字重 `850`。
- 卡片标题：`16px - 18px`，字重 `800 - 850`。
- 正文与表单：`14px`，字重 `600 - 700`。
- 辅助信息：`12px - 13px`，字重 `600 - 650`。
- 字间距保持 `0`，不使用负字距。

## 间距与圆角

- 基础间距使用 `8 / 12 / 16 / 20 / 24 / 32`。
- 普通按钮和输入框圆角：`8px`。
- 卡片圆角：`8px`，保持克制。
- 弹窗外层可用 `12px`，内部卡片仍用 `8px`。
- 页面区块之间至少 `24px`，卡片内部常用 `14px - 18px`。

## 组件规范

### 按钮

- 主按钮：蓝底白字，用于提交、保存、确认。
- 次按钮：白底蓝字或深蓝字，用于取消、刷新、返回。
- 危险按钮：白底红字红边，用于删除、清空。
- 按钮高度优先使用 `32px / 36px / 44px / 54px` 四档。
- 有明确图标时优先使用图标加文字。

### 输入框

- 输入框背景为白色，边框使用 `#dbe5f4`。
- 高度常用 `42px - 44px`，搜索框可用 `54px`。
- 文本域保持足够高度，需求说明不少于 `140px`。
- placeholder 使用次级灰蓝色，不要纯灰。

### 弹窗

- 弹窗应有明确标题区、辅助说明、主体内容、底部操作区。
- 弹窗宽度优先控制在 `640px - 760px`。
- 弹窗内部避免大面积原生表单堆叠，可用浅蓝说明区或卡片分组。
- 底部按钮右对齐，主操作在最右。

### 卡片

- 使用白底、`#e1e9f6` 边框、轻阴影。
- 标题、状态、时间要形成清晰层级。
- hover 可轻微上浮，但不要大幅动画。

## 门户侧当前优先落地范围

1. 需求反馈弹窗。
2. 通知弹窗。
3. 个人信息弹窗。
4. 修改密码弹窗。
5. 应用卡片与筛选区细节。

后台管理界面先不跟随本规范调整，后续单独整理。
