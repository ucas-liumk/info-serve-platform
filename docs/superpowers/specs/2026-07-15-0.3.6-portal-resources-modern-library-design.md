# 0.3.6 资料共享现代资料库设计

## 1. 背景与已选方向

本设计是 0.3.6 的第二个页面视觉增量。用户通过可点击原型选择 A“现代资料库”，并补充确认网格视图和列表视图都要优化。

当前资料共享页功能完整，已经具备搜索、栏目筛选、条件筛选、网格/列表切换、分页、预览、下载、收藏、上传和我的资料，但存在以下界面问题：

- 1440px 下主内容仍使用四列网格，资源卡宽度不足，标题、元数据和操作按钮互相挤压。
- 左侧栏目、顶部搜索、条件工具条和资源卡使用多套颜色、圆角、阴影及字重，视觉语言不统一。
- 页面同时出现“资源共享”和“资料共享”，违反 BC 术语统一要求。
- 列表视图仍是纵向卡片列表，没有稳定表头和列关系，不具备高效扫描体验。
- 小屏幕把完整栏目树放在内容之前，核心资料列表出现过晚。
- index.vue 当前 822 行，超过仓库单文件 800 行硬限。
- 页面仍有大量硬编码颜色、非标准字重和超出 elevation 规范的阴影。

## 2. 目标与非目标

### 2.1 目标

- 形成与门户首页一致的克制蓝白、低阴影、令牌化视觉体系。
- 搜索成为页面第一任务，筛选和栏目为辅助，资料内容保持主视觉。
- 网格视图解决溢出并提高卡片可读性；列表视图形成真正的资料表格。
- 桌面、平板、手机均不横向溢出，移动端优先显示搜索和资料结果。
- 保留现有接口、权限、路由、上传、预览、下载、收藏和分页行为。
- 拆分 index.vue，使所有修改后的 Vue 文件均不超过 800 行。

### 2.2 非目标

- 不采用原型 B 的推荐榜单、热门排行和门户运营区。
- 不把页面改成原型 C 的默认文件管理器，也不取消网格视图。
- 不修改后端、数据库、栏目数据结构、资源字段或预览引擎。
- 不修改图片资产和 /VERSION。

## 3. 页面骨架

### 3.1 桌面端

页面继续使用左侧栏目 + 右侧主内容结构：

- 左侧宽度由 276px 收敛为约 220px，保持 sticky，但降低品牌卡和筛选面板的视觉重量。
- 右侧从上到下依次为资料工具栏、条件工具条、已选分类 chips、网格或列表结果、分页。
- 页面外边距、区块间距只使用设计系统允许的 4/8/12/16/24/32/48/64 档位。

### 3.2 资料工具栏

从 index.vue 抽出 ResourceHeader.vue，负责：

- 页面标题和结果摘要。
- 搜索输入框与搜索按钮。
- 通知入口。
- 上传资料和我的资料入口。

搜索按钮是本屏唯一 primary。上传资料使用 secondary outline，我的资料使用 tertiary ghost；通知保持统一图标按钮。卡片和表格中的预览、下载、收藏均不得使用实心 primary。所有按钮使用现有高度、圆角、焦点和动效令牌。

### 3.3 左侧栏目

- 品牌统一写“资料共享”，图片 alt 同步为“资料共享”。
- 品牌卡、返回首页和筛选面板使用统一 radius-md、neutral 边框和至多 shadow-sm。
- 桌面保留栏目树。
- 980px 以下改为内联 disclosure：显示紧凑的“资料栏目/筛选”触发按钮，栏目树默认收起，可显式展开；触发按钮必须维护 aria-expanded 和 aria-controls，受控面板使用稳定 id，收起时使用 hidden 从可访问树中移除。
- 移动端展开后把焦点移动到第一个筛选控件；Escape 收起并把焦点归还触发按钮；断点返回 980px 以上时自动恢复桌面常显状态，不使用 dialog 或焦点陷阱。
- 分组展开与分类勾选拆成清晰的独立交互目标，避免 button 语义互相嵌套。

## 4. 网格视图

### 4.1 栅格

- 大于等于 1200px：三列。
- 768–1199px：两列。
- 小于 768px：单列。
- 使用 minmax(0, 1fr)，任何断点不得横向滚动。

### 4.2 资源卡

- 保留文档缩略图、标题、元数据、预览、下载和收藏。
- 标题最多两行，使用平衡换行或两行截断；完整标题通过 title 或可访问名称获得。
- 元数据允许合理换行，不使用 nowrap 把内容推到卡片外。
- 操作区固定在卡片底部，同一行卡片等高。
- 桌面操作目标至少 36px，移动端至少 44px。
- 卡片静态至多 shadow-sm，hover 至多 shadow-md；reduced-motion 下取消位移。
- 卡片、按钮、缩略图边框和状态颜色全部使用现有 --ip-* 令牌或其 color-mix 派生，不新增硬编码颜色。

## 5. 列表/表格视图

### 5.1 桌面表格

ResourceList.vue 改为原生 `<table>`，使用 `<thead>`、`<tbody>` 和带 `scope="col"` 的 `<th>`，列为：

1. 资料名称与简介
2. 分类/类型
3. 文件大小
4. 浏览/下载数据
5. 发布时间
6. 操作

表头使用 neutral-100 表面和 neutral-600 文字。数据行不使用独立大阴影卡，而以行分隔线和轻微 hover 表面区分。资料名称可截断但必须提供完整 title；发布时间统一格式化为 `YYYY-MM-DD HH:mm`，使用固定宽度且不得截断，不伪造缺失日期。

### 5.2 操作

- 每行保留预览、下载和收藏。
- 主次层级统一：预览为 secondary/tertiary 文字操作，下载和收藏为次级；收藏状态同时使用图标和文字，不只依赖颜色。
- 行点击可以继续预览，但必须同时提供可键盘访问的显式预览按钮，不能把整行点击作为唯一入口；行内按钮使用 stop 阻止重复触发。
- 网格/列表切换按钮使用 aria-pressed 或等价选中语义，图标按钮提供 aria-label。

### 5.3 响应式

- 768–1199px 隐藏低优先级列或把元数据合并到名称列，但保留名称、类型和操作。
- 小于 768px 时保留原生表格语义，视觉上隐藏表头，把每行布局成紧凑资料条目；各单元格通过 `data-label` 呈现字段标签，名称在上，元数据换行，操作位于底部；不得出现横向表格滚动。
- 移动端按钮高度至少 44px。

### 5.4 分页

- 768px 以上保留 `total, sizes, prev, pager, next, jumper` 完整分页。
- 小于 768px 改为 `prev, pager, next`，隐藏 total、sizes 和 jumper，并按可用宽度收敛 pager 数量；分页必须可键盘访问，触控目标至少 44px，375px 不得横向溢出。
- 页码与 page-size 变化继续调用现有处理函数；切换视图不得重置当前筛选，超出页数时沿用既有纠正逻辑。

## 6. 条件工具条与状态

### 6.1 工具条

- 保留排序、资料类型、上传时间、文件大小四个筛选条件。
- 统一 label、select 高度和间距，避免四组控件在窄屏互相挤压。
- 总数与视图切换靠右；窄屏按“筛选条件 → 总数/视图切换”分行。
- 当前视图不能只用颜色表达，必须有 aria-pressed 和稳定焦点态。

### 6.2 五态

- 加载：请求不足 300ms 不闪烁反馈；超过约 300ms 时显示与当前视图结构对应的骨架屏。网格骨架保持 3/2/1 列和卡片高度，表格骨架保持表头、固定列宽和行高；请求期间及视图切换时布局不得塌陷或跳动，不再仅依赖 `v-loading` spinner。
- 空数据：显示“暂无资料”并提供清除筛选或上传资料的恢复路径。
- 请求失败：结果区显示错误说明和“重新加载”按钮；不能只有全局 toast。
- 无权限：沿用现有登录/权限处理，不伪装成空数据。
- 零值：栏目计数可以显示 0，但资料统计文案不得制造演示数据。

列表请求新增 `loadError`，栏目树新增 `categoryLoadError`。列表失败时保留旧数据并在结果区显示错误，首次加载无旧数据时显示完整错误恢复区；栏目刷新失败时保留旧栏目树，同时显示“栏目刷新失败”和重试按钮。重试调用现有 reload/reloadWithFacets/loadCategoryTree，不改变 API。

删除 `loadCategoryTree()` 和 `reloadWithFacets()` 中的静默 catch。列表和栏目分别使用 latest-request 序号保护：成功与失败都只有在序号仍为当前请求时才可写入数据、loading 或错误状态，旧请求的失败不得覆盖新请求的成功。

### 7.1 全局视觉令牌约束

所有本次修改文件均遵守以下规则：

- 不新增或以另一硬编码值替换 hex/rgb/rgba；颜色只使用现有 `--ip-*` 令牌或基于令牌的 `color-mix()`。
- 字号只使用设计系统文本角色令牌，字重只允许 400/500/600/700。
- 所有 margin/padding/gap 只使用 4/8/12/16/24/32/48/64 间距档位；功能图标只使用 16/20/24/32px。
- 卡片使用 radius-md，按钮与输入使用 radius-sm；静态阴影最多 shadow-sm，hover 最多 shadow-md。
- 默认按钮与输入高度为 36px，移动端核心按钮为 44px，表格内紧凑操作按钮为 32px。
- `design-audit-baseline.json` 只允许随硬编码指标真实下降而下调，不得用基线掩盖新增或替换的违规值。

### 7.2 术语统一范围

所有可见文案统一使用“资料”，包括但不限于：页面与侧栏“资料共享”、我的“资料”、工具条“资料类型”、筛选面板“资料栏目”、卡片 fallback“资料共享”，以及 index.vue 中按钮、空态、错误和消息提示。`MyResourcesDrawer.vue` 的“我的资源”同步改为“我的资料”，`categoryFacets.ts` 的空选标题改为“全部资料”并同步更新测试。仅接口名、类型名和后端既有字段保留 Resource 技术命名。

## 8. 数据流与行为保持

- 搜索仍由 keyword 输入，Enter、搜索按钮和清空触发 reloadWithFacets。
- 分类、类型、时间、大小和排序仍调用现有 change 方法。
- displayMode 继续决定网格或列表，不改变默认网格视图。
- 预览仍打开 InfoResourcePreview 新窗口。
- 下载、收藏、上传、我的资料、分页和登录检查保持现有调用链。
- 快速筛选请求序号和旧响应丢弃逻辑扩展到成功、失败、loading 三种状态；通过 `resourceRequestState.ts` 提供可独立测试的 latest-request guard。
- 视觉改造不得吞掉请求错误或增加静默 catch。

## 9. 可访问性与动效

- 搜索输入有可访问名称；图标按钮有 aria-label。
- 网格/列表切换报告选中状态。
- 分类折叠、分类勾选、移动 disclosure、卡片/行操作均可键盘访问。
- 焦点使用 --ip-focus-ring，不裸移除 outline。
- hover 不改变布局尺寸；reduced-motion 下取消非必要位移和过渡。
- 375px 下所有核心操作至少 44px，页面无横向滚动。

## 10. 验收标准

- 1440px 网格稳定为三列，任何标题、元数据和按钮不越出卡片。
- 列表视图有清晰表头和稳定列关系，不再呈现为一组高阴影卡片。
- 网格/列表切换后筛选、分页、收藏状态保持。
- 页面所有可见术语统一为“资料共享/资料”，包括我的资料抽屉。
- 左侧栏不压制内容；980px 以下栏目可收起且默认不阻塞结果。
- 375/768/1024/1440 均无横向溢出。
- 375px 分页只显示 prev/pager/next，翻页与桌面 page-size 行为正确，触控目标达到 44px。
- 网格和表格骨架、空数据、列表失败、栏目失败及重试均可见且可操作；旧失败不会覆盖新成功。
- index.vue 和所有修改文件均不超过 800 行。
