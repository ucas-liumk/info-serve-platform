# 0.3.6 资料共享文件预览页沉浸式工作台设计

**状态：已确认**  
**确认日期：2026-07-15**  
**适用页面：`/portal/resources/preview/:resourceId`**  
**设计方向：A · 沉浸阅读工作台，高对比操作色阶**

## 1. 背景与目标

现有文件预览页同时常驻 PDF 阅读器、392px 功能面板、七个功能磁贴与多层工具栏。右栏中四个“即将上线”能力长期占位，实际可用的“我的笔记 / 交流互动 / 文件信息”反而不够直接；右栏收起后仍占 96px。工具图标与按钮使用偏浅的灰蓝色，在长时间阅读场景下辨认成本较高。

本次优化目标：

1. 让文档正文成为页面的绝对主角，桌面端默认只为功能轨保留 56px 左右空间。
2. 把文件级操作、PDF 阅读操作与资料协作能力拆成清晰的三层。
3. 使用设计系统中更高对比度的 neutral / primary / danger 令牌，提高图标和按钮可辨识度。
4. 保留现有下载、关闭、PDF 阅读、划词引用、笔记、公开交流、文件信息与阅看记录能力，不新增后端接口。
5. 拆分超出 800 行红线的 `ResourcePreviewContextPanel.vue`，让功能区可维护、可测试。

## 2. 信息架构

页面由四部分组成：

1. **文件顶栏**：返回资料列表、文件类型、标题、基础元数据、下载原文件、关闭预览。
2. **阅读操作条**：拖拽、连页、单页、双页连续、旋转、引用到笔记、日夜模式、全屏。
3. **文档舞台**：PDF / iframe / 图片 / 音视频 / 加载 / 空态 / 错误态的主要展示区。
4. **功能轨与抽屉**：常驻紧凑功能轨；“我的笔记 / 交流互动 / 文件信息”按需打开抽屉。

“OCR 识别 / 语音朗读 / 智能摘要 / 思维导图”本版本不再作为主界面禁用磁贴展示。它们没有可用行为，不应抢占阅读空间；后续接入时再按真实功能进入功能轨。

## 3. 桌面端布局（>= 1200px）

### 3.1 默认状态

- 页面占满 `100dvh`，外层仅保留设计令牌允许的紧凑边距。
- 顶栏横跨阅读区和功能轨，文件标题单行省略，完整标题通过原生 `title` 或 tooltip 获取。
- 文档舞台占据除功能轨以外的全部剩余宽度。
- 功能轨固定约 56px，按顺序放置：我的笔记、交流互动、文件信息；底部放置展开/收起按钮。
- 功能轨按钮桌面点击目标不小于 36px，图标 20px；hover 显示文字提示。
- 默认激活项可以沿用用户本次会话的上次选择；无记录时不自动打开抽屉，保持沉浸阅读。

### 3.2 抽屉展开状态

- 点击功能轨项目后，从右侧展开 320–392px 抽屉，不挤压到小于 650px 的阅读区。
- 再次点击当前项目或点击收起按钮关闭抽屉；切换项目时在同一抽屉内替换内容。
- 抽屉使用 `--ip-shadow-lg` 与边框表达层级，320ms 以内完成进出；遵循 `prefers-reduced-motion`。
- 抽屉头部显示当前功能名称和关闭按钮，不重复文件标题与下载按钮。
- 抽屉可保留当前拖拽调宽能力，宽度范围继续由 `clampPanelWidth` 约束；收起后不保留大面积空栏。

### 3.3 文件顶栏

- 顶栏高度 56px 左右，使用 neutral-0 背景和 neutral-200 分隔线。
- 左侧返回按钮为 secondary 图标按钮；文件类型用 primary soft 标签。
- 标题使用 neutral-900、700 字重；副信息使用 neutral-500，内容为分类、文件大小、阅读进度提示中的可用项。
- “下载原文件”是本页唯一 primary 实心按钮。
- “关闭预览”是 danger 语义的描边图标按钮；关闭不是数据破坏操作，因此不使用实心 danger。

### 3.4 阅读操作条

- 保留现有全部 PDF 控制能力，不更改 EmbedPDF 交互契约。
- 默认图标与文字使用 `--ip-neutral-600`，不再使用低对比度弱灰。
- hover 使用 primary-50 / primary-700；选中态使用 primary-100 背景、primary-700 图标与文字、primary-200 边框。
- 每个控件必须同时有图标与可访问名称；窄桌面可优先隐藏部分文字，但不得只靠颜色表达选中状态。
- 与 EmbedPDF 内建工具条的重复项不再额外增加；本次只精简视觉和层级，不移除已有能力。

## 4. 功能抽屉内容

### 4.1 我的笔记

- 顶部说明“私有，可选择公开”，正文编辑器为首要内容。
- “保存笔记”在抽屉内部可作为 primary；它与页面顶栏 primary 不同时出现在同一操作上下文，视觉上以抽屉范围为独立容器。
- 支持新建、编辑、取消、删除、私有/公开切换与“分享到交流互动”。
- PDF 划词引用后自动打开“我的笔记”抽屉，并把格式化引用追加到编辑器。
- 保存中按钮禁用并显示明确加载状态，防止重复提交。

### 4.2 交流互动

- 保留公开讨论的发布、编辑、删除和列表能力。
- 当前用户内容保留“我的”标识；删除继续使用确认弹窗。
- 发布中按钮禁用并显示加载状态。

### 4.3 文件信息

- 展示资料分类、原始文件、大小、发布人、发布时间、阅看/下载统计与阅看记录。
- 标签列和内容列有稳定对齐，长文件名可换行，不截断关键时间字段。

## 5. 高对比色阶

全部颜色必须来自 `tokens.scss`：

| 场景 | 令牌 |
|---|---|
| 标题与关键文字 | `--ip-neutral-900` / `--ip-neutral-800` |
| 默认操作图标与文字 | `--ip-neutral-600` |
| 次级说明 | `--ip-neutral-500` |
| 默认边框 | `--ip-neutral-200`，重要控件可用 `--ip-neutral-300` |
| 主操作与选中态 | `--ip-primary-600/700` + `--ip-primary-50/100/200` |
| 关闭按钮 | danger base / soft / border 令牌 |

正文与控件文字对比度必须达到 WCAG AA；禁用态才允许使用 neutral-400。不得新增硬编码颜色、字号、圆角或阴影。

## 6. 响应式

### 6.1 768–1199px

- 顶栏保留标题与主要文件操作，副信息可隐藏。
- 功能轨继续常驻；抽屉以覆盖式出现，不持续挤压阅读区。
- 阅读操作条允许横向滚动或按逻辑分组收敛，不能裁切控件。

### 6.2 < 768px

- 点击目标统一不小于 44px。
- 顶栏只保留返回、文件类型/标题、下载与关闭；下载可收敛为带 aria-label 的图标按钮。
- 功能入口改为底部三项操作栏，点击后使用全宽底部抽屉或全屏层展示内容。
- PDF 阅读区保持可滚动，不把右侧轨硬塞进窄屏。

## 7. 状态与错误处理

- **加载**：文件信息与预览地址加载时显示骨架或明确加载文案；PDF 查看器保留加载遮罩。
- **空态**：缺少预览地址时说明原因，并提供“返回资料列表”和“下载原文件”（若可用）。
- **不支持**：OFD 或不支持格式明确提示下载查看，不留空白舞台。
- **错误**：预览失败时显示错误说明、重试和下载；不能只弹 toast 后留下空白。
- **无权限**：沿用全站 403 视图或接口拦截器规范，不伪装成预览失败。

## 8. 组件拆分

`ResourcePreviewContextPanel.vue` 当前超过 800 行，必须拆分。建议边界：

- `ResourcePreviewWorkbench.vue`：功能轨、抽屉容器、状态切换、调宽。
- `ResourcePreviewNotesPanel.vue`：我的笔记编辑与列表。
- `ResourcePreviewDiscussionPanel.vue`：公开交流编辑与列表。
- `ResourcePreviewInfoPanel.vue`：文件元数据和阅看记录。
- 可复用的 API 加载与写操作放入 `useResourcePreviewNotes.ts`，避免两个面板复制刷新逻辑。
- `preview.vue` 只负责资源加载、预览模式、文件顶栏与舞台编排。

拆分后每个 `.vue` / `.ts` 文件均不得超过 800 行；新增子组件必须显式 import。

## 9. 交互与可访问性

- 所有图标按钮提供 `aria-label` 或可见文本。
- 所有交互元素使用全局 `:focus-visible` 焦点环；不可移除焦点态。
- 拖拽调宽手柄保留 separator 语义、双击复位与 pointer capture。
- Esc 在抽屉展开且未进入确认弹窗时优先关闭抽屉；全屏状态由浏览器 Fullscreen API 处理。
- 禁止仅用颜色表示激活、错误或公开/私有状态。

## 10. 验收标准

1. 1440px 下默认功能区宽度由现有约 392px 降至约 56px，文档舞台明显增宽。
2. 主界面不再显示四个“即将上线”磁贴。
3. 三个现有功能可从功能轨进入，切换、收起、调宽和数据操作均正常。
4. 顶栏、工具条、功能轨默认图标与文字的可辨识度明显高于旧版，禁用态除外。
5. PDF 阅读、翻页、旋转、布局、划词引用、夜间、全屏、进度记忆无回归。
6. 图片、iframe、音视频、不支持格式、加载与错误状态布局正常。
7. 1440 / 1024 / 768 / 375px 无横向裁切；移动端点击目标不小于 44px。
8. 组件文件均不超过 800 行。
9. `npm run design:audit`、`npm run build:prod` 和相关单元测试通过。
10. 使用 `deploy/scripts/ui-capture.py` 或 Playwright 保存改造前后截图，至少覆盖默认态、笔记抽屉展开态与移动端。

## 11. 非目标

- 不实现 OCR、语音朗读、智能摘要、思维导图。
- 不修改资料、笔记、交流或阅看记录后端接口。
- 不替换 EmbedPDF 库，不重写 PDF 渲染引擎。
- 不改变资料权限、收藏、下载统计与审计规则。
