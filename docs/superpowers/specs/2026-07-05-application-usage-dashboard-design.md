# 应用态势看板设计

## 背景

本项目当前尚未承载数字化转型项目、任务办理、问题闭环等业务，因此第一版不做完整“数字化转型驾驶舱”。本次交付聚焦“当前系统应用情况洞察看板”，帮助单位领导快速了解平台是否被用起来、哪些模块和应用活跃、资料是否流动、部门覆盖是否均衡、需求反馈是否积压。

## 目标

- 提供一个面向领导投屏和日常查看的应用态势入口。
- 用当前系统已有数据形成可解释的运营洞察，不硬造转型评分。
- 展示模块活跃、资料流动、应用热度、部门覆盖、需求反馈响应和异常信号。
- 页面视觉要比普通后台报表更有态势感，但仍保持政务系统的清晰、稳重和可读性。

## 非目标

- 不新增数字化转型项目管理、任务办理、督办闭环、问题闭环业务。
- 不引入新的大屏框架或图表库。
- 不做全屏黑蓝炫酷大屏，不牺牲可读性追求装饰效果。
- 不绕过限界上下文直接跨域裸查业务表。

## 技术方案

### 前端

- Vue 3 + TypeScript。
- Element Plus 作为基础交互控件。
- ECharts 6 用于趋势、排行、部门热力和反馈状态图表。
- SCSS + 现有 `--ip-*` 设计令牌控制颜色、间距、边框和阴影。
- 新页面入口建议为 `/portal/usage-dashboard`，展示名为“应用态势”。

### 后端

- 在 `ruoyi-portal` 的 `kernel` 下新增应用态势聚合接口。
- 建议接口：`GET /portal/usage-dashboard/overview`。
- 内核只消费各 BC 的 service 接口：
  - AppCenter 提供工具总量、使用排行、收藏/使用统计、需求反馈状态统计。
  - Resources 提供资料总量、资料浏览/下载排行、资料流动统计。
  - Forum 提供论坛主题/回复/活跃用户统计。
  - System 或可用登录/用户信息提供部门覆盖基础口径。
- 如第一版某些部门热力口径缺失，接口返回空集合或降级数据，前端明确显示“暂无足够数据”，不伪造部门评分。

## 页面结构

### 1. 顶部区

- 标题：`系统应用情况洞察看板`
- 副标题：`看模块活跃、资料流动、应用热度、部门覆盖与反馈响应`
- 过滤信息：时间范围、数据范围、刷新时间。

### 2. KPI 行

首屏顶部展示 6 个核心指标：

- 可用工具：已上架、可访问的应用数量。
- 资料资产：公开可用资料数量。
- 活跃用户：周期内发生登录、使用工具、浏览/下载资料、发帖/回帖等行为的用户数。
- 部门覆盖：周期内有应用行为的部门数 / 可见部门数。
- 使用次数：工具使用 + 资料浏览下载等主行为合计。
- 待处理反馈：应用需求反馈中待处理数量。

### 3. 第二排运营面板

三列并排：

1. 模块活跃趋势
   - 展示工具即用、资料共享、服务论坛、通知/消息等模块在周期内的活跃量。
   - 优先使用柱状或折线趋势，数据不足时用横向排行。

2. 资料流动排行
   - 展示浏览/下载/收藏综合热度最高的资料。
   - 以资料标题、分类、热度值呈现。

3. 应用热度排行
   - 展示使用量最高的应用。
   - 以应用名称、分类、使用次数和趋势提示呈现。

### 4. 中间主视图

部门 × 模块应用热力：

- 行：部门。
- 列：工具、资料、论坛、收藏、反馈。
- 单元格：周期内行为量或标准化热力。
- 如果部门归属或行为日志不足，显示数据不足说明，并保留未来接入位置。

### 5. 底部区

两列：

1. 需求反馈响应统计
   - 展示本月反馈数、待处理、处理中、已处理。
   - 展示反馈类型分布：新应用、改进建议、问题反馈。
   - 名称采用“响应统计”，强调处理状态而不只是数量。

2. 低活跃与异常信号
   - 长期无人使用应用。
   - 低活跃部门。
   - 待处理反馈积压。
   - 论坛互动明显下降等可由当前数据支撑的信号。

## 视觉设计

- 风格：稳重政务科技感。
- 背景：浅色为主，可使用极浅蓝灰底和细网格/边框层次，不使用大面积紫色渐变或夸张光效。
- 卡片：8px 以内圆角，边框 + 轻阴影，避免卡片套卡片。
- 图表：蓝绿主色用于主数据，琥珀/红色只用于风险提示。
- 字体：沿用项目字体栈和设计令牌，不单独引入外部字体。
- 响应式：桌面优先，兼容 1366、1440、1920 宽度；窄屏改为纵向堆叠。

## 数据与降级

- 所有指标必须注明口径来源。
- 无数据时展示空态，不显示伪造数字。
- 部门热力第一版允许降级：若无法准确按部门归集，只显示“部门覆盖口径待接入”，其他模块仍正常展示。
- 需求反馈按现有 `app_demand` 状态统计。

## 验收标准

- 新入口能从门户访问应用态势页面。
- 页面首屏包含 KPI、三列运营面板、部门热力、需求反馈响应统计和异常信号。
- 后端接口返回结构化聚合数据，缺失口径有明确空集合或空态字段。
- 遵守 BC 边界：内核聚合只依赖 service 接口，不依赖内容 BC 的 mapper/domain/impl。
- 必跑验证：
  - `mvn -o -ntp -Pdev -DskipTests -pl ruoyi-modules/ruoyi-portal -am compile`
  - `mvn -pl ruoyi-modules/ruoyi-portal -am -DskipTests=false test`
  - `cd plus-ui && npm run build:prod`
  - `cd plus-ui && npm run design:audit`
  - 视觉页截图检查。
