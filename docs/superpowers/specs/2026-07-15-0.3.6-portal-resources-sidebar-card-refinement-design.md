# 0.3.6 资料共享栏目与文件卡片精修设计

## 1. 背景与决策

本设计是 `2026-07-15-0.3.6-portal-resources-modern-library-design.md` 的视觉精修增量，不改变已完成的资料共享功能、接口、表格视图、错误恢复和移动筛选行为。

用户验收当前网格页后指出两个主要问题：左侧栏目文字显示不全，文件卡片内部空白过多、缺少设计感。经过四轮可点击原型迭代，最终确认 V4“贯穿式栏目底板 × 紧凑工具栏 × 四列文档卡片”。

本次确认的关键决策：

- 左侧栏目由悬浮的小筛选卡改为贯穿右侧内容高度的连续导航底板。
- 1440px 桌面视口下，右侧网格由三列调整为四列。
- 顶部“全部资料”区域由约 110px 的横幅收敛为约 76px 的单行工具栏。
- 文件卡由约 184px 收敛为 132–152px，封面、标题、元数据和操作重新分配空间。
- 设计感来自连续层级、文档册页比例、排版和单一蓝色强调，不新增装饰性渐变、大阴影或非令牌颜色。

### 1.1 对主规格的覆盖关系

本增量规格明确替代主规格以下条款：

- 主规格 §3.1 的 220px 桌面左栏，替换为本规格 §3.1 的 264px 左栏与两层贯穿底板。
- 主规格 §3.3 与 §10 的 980px disclosure 条款，替换为本规格 §4.2 与 §9 的 1120px disclosure 条款。
- 主规格 §4.1 的 3/2/1 网格列数，替换为本规格 §5.1 的 4/3/2/1 列数。
- 主规格 §6.2 的对应网格骨架列数，替换为本规格 §5.3 的 4/3/2/1 骨架规则。
- 主规格 §10 的“1440px 三列”验收，替换为本规格 §9 的“1440px 四列”验收。

除以上明确列出的视觉条款外，主规格其余功能、状态、表格、错误恢复、可访问性和验证要求继续有效。

## 2. 范围与非目标

### 2.1 修改范围

- `plus-ui/src/views/portal/resources/index.vue`
- `plus-ui/src/views/portal/resources/components/ResourceHeader.vue`
- `plus-ui/src/views/portal/resources/components/ResourceSidebar.vue`
- `plus-ui/src/views/portal/resources/components/ResourceFilterPanel.vue`
- `plus-ui/src/views/portal/resources/components/ResourceCard.vue`
- `plus-ui/src/views/portal/resources/components/ResourceLoadingSkeleton.vue`
- 与上述组件直接对应的测试；如网格样式在独立组件中，则只改该组件。
- `plus-ui/design-audit-baseline.json` 仅在违规指标真实下降时同步下调。

### 2.2 非目标

- 不修改后端接口、栏目数据结构、文件字段、数据库或权限逻辑。
- 不改变表格视图的列定义、排序、分页、搜索、筛选、预览、下载和收藏行为。
- 不修改 `/VERSION`，不创建单页 PR。
- 不引入新字体、图标库、图片资产、硬编码颜色或新的视觉令牌。
- 不把真实缩略图裁切成横幅，不隐藏预览、下载或收藏入口。

## 3. 桌面页面骨架

### 3.1 左右布局

- 桌面左栏宽度由约 220px 调整为 264px；右侧继续使用 `minmax(0, 1fr)`。
- 左右间距使用现有 16px 或 24px 间距令牌，不新增任意间距值。
- 左栏采用两层结构：外层连续白色表面作为 grid item，使用 stretch 对齐并延伸到右侧页面内容底端；内层承载品牌、返回首页、栏目树和错误状态，使用 sticky 与视口高度约束。
- 内层使用纵向 flex/grid：品牌与返回首页固定在上方，栏目树是唯一可滚动区域，栏目错误和清除筛选位于该滚动区域的可见末端。外层不得设置 `max-height` 或 `align-self: start`；现有 sticky、`max-height: calc(100vh - 48px)` 和滚动约束迁移到内层，禁止外层与内层同时滚动。
- 左栏只保留一个连续外层表面。品牌、返回首页、栏目树不再各自表现为三张同权重悬浮卡片。

### 3.2 紧凑资料工具栏

- 桌面高度目标为 76px，允许因系统字体产生少量自然偏差，但不得恢复为大横幅。
- 单行分为三段：左侧“全部资料 + 当前数量”，中间搜索框，右侧上传资料、我的资料和用户/通知入口。
- 左侧标题继续显示当前分类标题；未选择分类时显示“全部资料”，选择一个分类时显示该分类名称，选择多个分类时沿用现有汇总标题。当前结果数量继续来自列表请求 `total`。
- “资料共享”小标签可隐藏或降为辅助文案，不能单独占一行。
- 搜索仍是唯一 primary；上传资料和我的资料保持 secondary/tertiary。
- 1121–1320px 可适当压缩三段宽度；空间不足时允许搜索换到下一行，但不得遮挡或横向滚动。1120px 以下左栏已切换为顶部 disclosure，工具栏可以使用主内容完整宽度。

## 4. 左侧栏目导航

### 4.1 信息层级

- 顶部显示“资料栏目”和总量/收起操作。
- “全部资料”是一级入口，使用浅蓝选中表面和栏目图标。该入口不展示推算总数，避免把不可相加的栏目分面计数伪装成全部资料数量。
- 分组标题使用箭头、标题和延伸分隔线，强化“组 → 子栏目”的关系。
- 子栏目用细竖向导轨表达层级；当前栏目额外使用左侧蓝色短条和浅蓝表面，不只依赖文字颜色。
- 数量使用固定宽度或 `flex: none` 的右侧胶囊，采用 tabular number，不能挤压栏目名称。

### 4.2 完整文字与交互

- 子栏目名称默认允许两行显示，禁止对正常中文分类使用单行省略号。
- 两行仍不足时才使用截断，并通过 `title` 或现有 tooltip 提供完整名称。
- 分类名称区域使用 `min-width: 0; flex: 1`，数量区域独立；不得让复选框、箭头、文字和数量互相争夺同一行宽度。
- 分组展开按钮和分类选择控件继续保持独立语义，不嵌套 button。
- 可交互目标桌面至少 36px，移动端至少 44px；保留可见焦点环、`aria-expanded`、`aria-controls` 和选中状态。
- 1120px 以下继续采用已实现的 disclosure，不强制显示贯穿式桌面底板。
- 为保证 1024px 三列卡片可用宽度，桌面侧栏/disclosure 断点由 980px 调整为 1120px；1120px 及以下使用顶部 disclosure，1121px 及以上显示贯穿式左栏。
- 点击“全部资料”只清空 `selectedCategories`，保留关键词、排序、资料类型、上传时间和文件大小筛选；无分类选择时该入口为选中态。
- 每个分组内容区域新增稳定 id，分组展开按钮通过 `aria-controls` 与之关联；不能把这项能力写成已经存在。

## 5. 四列网格与文件卡片

### 5.1 响应式列数

- 视口宽度大于 1320px：四列，确保 1440px 验收视口稳定显示四列。
- 981–1320px：三列；1120px 及以下左侧栏切换为顶部 disclosure，为 1024px 三列释放完整主内容宽度。
- 768–980px：两列。
- 小于 768px：单列。
- 网格必须使用 `repeat(n, minmax(0, 1fr))`，所有断点不得产生横向滚动。
- 若现有页面容器宽度与视口断点不完全一致，可用等价容器查询实现，但 375/768/1024/1440 的最终列数必须分别为 1/2/3/4。

### 5.2 卡片结构

- 桌面四列卡片目标高度为 132–152px，以最长合法标题、三位数统计和三个操作仍不重叠为准；不得为了命中 132px 强行压缩可读性，也不使用造成大块空白的 `minmax(0, 1fr)` 内容行。
- 卡片采用约 74×106px 的文档舞台，真实缩略图保持完整比例并居中，不裁掉正文；无缩略图时继续使用统一的文件类型封面。
- 右侧内容顺序固定为：栏目 overline、最多两行标题、文件大小/浏览/下载元数据、操作区。
- 标题使用 `min-width: 0` 和两行截断，完整标题通过 `title` 或可访问名称获得。
- 操作区紧跟内容或通过适度 `margin-top: auto` 对齐，但卡片不得依赖 184px 以上固定高度制造空白。
- 所有预览、下载和收藏按钮始终提供图标、动态 `aria-label` 与 `title`。卡片内容宽度大于等于 280px 时预览和下载显示文字；四列卡内容宽度不足 280px 时允许只显示图标，以容纳三个操作且不溢出。不能仅依赖 hover tooltip 提供可访问名称。
- 静态卡片至多 `shadow-sm`，hover 至多 `shadow-md`；只允许轻微 transform/边框/表面变化，并在 `prefers-reduced-motion` 下取消位移。

### 5.3 三列、两列与单列

- 三列允许封面回到稍宽尺寸并显示预览/下载文字，但不得出现同一组件在断点间明显跳高。
- 两列和单列优先保证标题、元数据与操作可读；移动端操作目标至少 44px。
- 375px 单列不得因为三个操作按钮或长标题产生横向溢出。
- 骨架屏必须复用相同的 4/3/2/1 列规则和卡片比例，避免加载完成时布局跳动。
- 边界样例必须覆盖：最长合法中文标题、三位数浏览/下载统计、最长文件大小文本和三个操作同时存在；该样例在 1024px 三列与 1440px 四列均不得重叠或溢出。

## 6. 视觉令牌与一致性

- 所有颜色只使用现有 `--ip-*` 令牌或其 `color-mix()` 派生。本次触及的 `ResourceCard.vue` 卡片与 fallback 封面样式必须同步清理现存 `#fff`、硬编码渐变、任意字号和 `0.16s ease` 等硬编码；颜色、字号、圆角、阴影与动效全部映射到 `tokens.scss` 现有令牌。
- 使用现有 `radius-sm`/`radius-md`、`shadow-sm`/`shadow-md`、焦点和动效令牌。
- 字号只使用现有文本角色令牌，字重仅允许 400/500/600/700。
- margin、padding、gap 仅使用仓库允许的 4/8/12/16/24/32/48/64 档位。原型中的 10/11/13px 仅表达视觉比例，生产实现必须映射到合法令牌档位。
- 图标继续使用项目现有 Element Plus 图标，禁止 emoji 或新图标包。
- 左侧和卡片使用同一边框、表面、圆角与强调蓝层级，避免再次形成多套视觉语言。

## 7. 组件边界与行为保持

- `ResourceSidebar.vue` 负责外层连续底板、内层 sticky 容器、品牌、返回首页、移动 disclosure 和错误区编排。
- `ResourceFilterPanel.vue` 负责栏目层级、展开、选择、数量与清除筛选，不把页面布局逻辑塞入筛选组件。
- `ResourceCard.vue` 负责文档舞台、资料信息和卡片操作；缩略图缓存、PDF 渲染和失败负缓存逻辑保持不变。
- `ResourceHeader.vue` 负责 76px 桌面工具栏和各断点重排；继续接收并显示现有动态标题和列表 `total`，不把标题逻辑移回 `index.vue`。
- `index.vue` 只调整页面 grid 和网格列断点，不回收已拆出的顶栏或请求状态逻辑。
- 所有修改后的 `.vue` 文件必须小于 800 行；若 `ResourceCard.vue` 因样式增长接近 800 行，应抽取纯展示子组件或复用现有样式，禁止机械堆叠。

## 8. 可访问性与状态

- 左栏当前项同时使用背景、短条和语义状态，不只依赖颜色。
- 长栏目文字在 200% 文本缩放下仍可获得完整内容；计数不得覆盖文字。
- 卡片整体点击预览可以保留，但显式预览按钮仍是键盘用户的稳定入口。
- 所有按钮保持可见焦点环；图标按钮有可访问名称。
- 加载、空态、列表失败、栏目失败和重试行为沿用既有实现，视觉精修不得吞掉错误或旧数据。
- 四列布局不改变分页、搜索、筛选或收藏状态。

## 9. 验收标准

- 1440px：左栏白色背景板从顶部延伸到右侧内容底端；右侧稳定四列。
- 1024px：左栏为顶部 disclosure，右侧三列，栏目文字和操作不溢出。
- 768px：两列，桌面左栏改为顶部 disclosure。
- 375px：单列，所有核心按钮至少 44px，无横向滚动。
- “技术文档与实施指南”“数据治理与数据标准”等长分类默认完整显示或可明确获得完整文本。
- 顶部“全部资料”工具栏桌面高度约 76px，搜索、上传和我的资料不被遮挡。
- 卡片桌面高度位于 132–152px；短标题卡片不再出现明显的大块无内容区域，最长合法标题、三位数统计和三个操作也不重叠。
- 真实 PDF/图片缩略图和 fallback 封面均保持清晰、比例一致，不拉伸或裁掉主体。
- 网格/列表切换、搜索、分类、四组筛选、分页、预览、下载、收藏、上传和我的资料行为保持。
- `npm run design:audit` 与 `npm run build:prod` 通过；本次触及的卡片/fallback 规则清除现存硬编码颜色、渐变、任意字号和动效值，而不只是保证基线不增长。
