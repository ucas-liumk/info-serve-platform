# 0.3.6 门户首页视觉平衡补充设计

## 1. 背景与结论

本设计补充首页右上角工具栏方案，处理工具栏变轻后出现的首页视觉重心偏上、统计区下方留白过多，以及模块卡片与统计卡片视觉重量接近的问题。

用户确认采用“整体下移 + 统计区降层级 + 背景柔化”的轻量统一方案。首页内容、六列模块结构、模块插画、统计指标和业务行为均保持不变。本次不引入 Bento 重排、额外装饰光效或新图片资产。

## 2. 设计目标

- 模块卡片和统计区作为同一个内容组整体下移，减少首屏下半部无效留白。
- 模块卡片保持主内容层级，统计区退为辅助数据带，不与服务入口争夺注意力。
- 统一两类卡片的圆角、边框、表面色和阴影体系。
- 让长城背景到浅色页面底部的过渡更早、更柔和，使统计区自然落入安静背景。
- 保持 375px、768px、1024px 和 1440px 下不横向溢出，并避免在移动端增加无意义的首屏空白。

## 3. 垂直节奏

只调整 ModuleGrid 的上边距。StatsBand 继续位于模块区之后，二者现有 24px 间距不变，因此会作为一个整体同步移动。

首页沉浸式首屏的模块起始位移不是普通组件间距。为避免继续写裸值，本次先在 tokens.scss 增加三个首页专用语义布局令牌，并在 design-system.md 的布局章节登记为仅限门户首页首屏使用的例外；普通 margin/padding 仍只能使用 4/8/12/16/24/32/48/64 八档：

- --ip-home-module-offset-canvas: 192px，用于大于 1460px。
- --ip-home-module-offset-wide: 152px，用于 1181–1460px；1440px 相对当前实现下移 40px。
- --ip-home-module-offset-desktop: 112px，用于 1024–1180px；相对当前实现下移 16px。
- 1023px 及以下只使用允许的 64px/48px 间距档；原 40px 收敛为 48px，不新增其他移动端纵向空白。

不得用 transform: translateY() 制造视觉位移；必须使用正常文档流间距，避免命中区域、滚动位置和视觉位置不一致。

## 4. 卡片层级统一

### 4.1 模块卡片

- 保留六列结构、现有尺寸、插画、标题和描述。
- 圆角从裸 16px 收敛为普通卡片令牌 --ip-radius-md。
- 静态阴影使用 --ip-shadow-sm，hover 时最多升为 --ip-shadow-md，符合卡片 elevation 上限。
- 卡片表面透明色改用现有 --ip-* 令牌派生的 color-mix()，不得新增十六进制、rgb() 或 rgba() 硬编码颜色。
- 模块卡片继续是完整 button，箭头仅作为视觉方向提示，不新增重复点击控件。
- 补齐清晰的 :focus-visible 焦点环；焦点反馈不得引发布局位移。

### 4.2 统计区

- 保持 24px 的模块区间距、现有指标、顺序和网格响应式行为。
- 圆角从裸 16px 收敛为 --ip-radius-md，与模块卡片一致。
- 移除常驻阴影，仅以 neutral 边框和表面区分；不得增加 hover 阴影。
- 表面色使用令牌派生的半透明 neutral 表面，与模块卡片同色系但透明度更高。
- 统计图标圆形从 52px 收敛为 48px，内部图标明确使用允许档位 24px；数据值和标签字号不变，避免损害读数效率。
- 分隔线降低视觉对比度，但仍使用现有 neutral 令牌派生，保证浅色背景下可辨识。

## 5. 背景衔接

- 保留长城背景图片、定位方式和整体品牌氛围。
- 仅调整首页现有白色渐变的停靠位置，让浅色过渡略早进入模块与统计区域。
- 不新增大面积遮罩面板，不模糊背景图片，不改变图片资产。
- index.vue、ModuleGrid.vue、StatsBand.vue 均不得新增或用另一组值替换出任何 hex/rgb/rgba。背景柔化只调整现有渐变 stop 百分比；组件表面和分隔线改动必须由 --ip-* 令牌通过 color-mix() 派生。
- 设计审计硬编码颜色基线只能下降，不能用“总数持平”掩盖硬编码颜色替换。

## 6. 动效与可访问性

- hover 抬升只用于可点击模块卡片；统计区不增加 hover 动效。
- 模块卡片和“更多服务”按钮必须具有可见键盘焦点。
- 在 prefers-reduced-motion: reduce 下关闭非必要位移和过渡。
- 保持现有 DOM 顺序和标题语义，不改变统计数据的屏幕阅读顺序。

## 7. 文件范围

允许修改：

- plus-ui/src/views/portal/home/index.vue
- plus-ui/src/views/portal/home/components/ModuleGrid.vue
- plus-ui/src/views/portal/home/components/StatsBand.vue
- plus-ui/src/assets/styles/tokens.scss
- docs/design/design-system.md
- 如设计审计指标下降，允许同步更新 plus-ui/design-audit-baseline.json，但禁止上调任何基线。

不修改后端、接口、数据库、路由、模块注册表、图片资产和 /VERSION。

## 8. 验收标准

- 1440px 下模块与统计区相对当前版本整体下移约 40px，统计区下方留白明显减少。
- 模块区与统计区仍保持 24px 间距，六列卡片等宽且无重排。
- 模块卡片是主层级，使用 sm 静态阴影和 md hover 阴影；统计区无常驻阴影且图标重量更轻。
- 背景由长城图向浅色底自然过渡，无新增大面板或明显色带断层。
- 375px、768px、1024px、1440px 无横向滚动、遮挡和异常首屏空白。
- 模块卡片键盘焦点清晰；reduced-motion 下不执行非必要位移动效。
- Windows 验收服务中首页接口无代理 500，模块入口和统计数据显示行为不变。

## 9. 验证与版本控制

实现与重型验证只在 Windows 独立工作目录执行。设计审计与构建必须先进入 plus-ui：

    chcp 65001 >nul
    cd /d E:\gallant-dev\active\info-serve-0.3.6-home\plus-ui
    npm run design:audit
    npm run build:prod

官方截图从 Windows 仓库根目录执行，并将前后结果放入不同目录：

    cd /d E:\gallant-dev\active\info-serve-0.3.6-home
    set UI_CAPTURE_BASE=http://127.0.0.1:17018
    set UI_CAPTURE_OUT=output\ui-capture-home-balance-before
    python deploy\scripts\ui-capture.py
    set UI_CAPTURE_OUT=output\ui-capture-home-balance-after
    python deploy\scripts\ui-capture.py

官方脚本覆盖 1440px 与 390px。另用浏览器响应式模式记录 768px 和 1024px 人工检查结果；四个宽度都检查背景过渡、统计区下方留白和横向滚动，1440px/390px 还需保留截图。键盘检查模块卡片及“更多服务”焦点态，并在系统 reduced-motion 开启后确认无非必要位移。若自动登录失败，记录失败原因并使用同 origin 的已登录浏览器完成等价截图，不得把登录页当作首页基线。

接口与行为冒烟使用当前 Windows 验收 origin http://192.168.8.10:17018：

    curl -i http://127.0.0.1:17018/dev-api/

期望代理到后端并返回业务 JSON（未登录时允许业务 401），不得出现 Vite proxy 500/ECONNREFUSED。浏览器中核对统计区完成加载或显示现有错误反馈，依次点击六个首页模块卡片并确认其原有路由/不可用提示没有变化。

最后检查 Windows Git 工作区仅包含预期文件。实现正常提交并推送当前 0.3.6 UI 分支作为版本控制检查点，但不为单个首页调整创建 PR；待所有页面样式完成后统一提交 0.3.6 UI PR。
