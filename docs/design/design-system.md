# 信息中心数智服务平台 · 设计系统规范

**版本 v1.0 · 状态：生效（构建门禁强制） · 适用：门户 / 管理后台 / 后续全部入口**

| 关键事实 | 说明 |
|---|---|
| 唯一正本 | 本文件。`docs/ui-ux-design-guidelines.md` 与 `plus-ui/DESIGN_GUIDE.md` 已废止，仅存指引头 |
| 生效机制 | 令牌层代码 `plus-ui/src/assets/styles/tokens.scss` 与本文一一对应；`npm run design:audit` 为构建门禁（棘轮机制，见 §9） |
| 变更流程 | 修改令牌 = 同时改本文与 tokens.scss，走分支评审合入；新增颜色/字号/圆角一律先立令牌后使用 |
| 术语基线 | 界面文案必须使用 [bounded-contexts.md](../architecture/bounded-contexts.md) §2 统一语言（如"资料共享/应用中心"），禁止同义词混用 |

## 1. 设计原则

1. **系统优先于页面。** 任何视觉决策先问"令牌里有吗"；没有就先提令牌评审，而不是就地发明。单页的"更好看"不构成偏离系统的理由。
2. **克制即专业。** 政企内网产品的可信感来自秩序：少色、少档、少装饰。默认不加渐变、不加大投影、不加动效，除非规范明确允许。
3. **状态完备才算完成。** 一个界面交付必须包含：默认、加载（骨架）、空数据、错误、无权限五态。只做"数据很多时好看"的界面视为未完成。
4. **低数据量下依然体面。** 内网系统长期处于少量数据状态。零值、单条记录、示例期指标都要有专门的呈现策略（§8.4），禁止"0 次 × N"裸奔。

## 2. 色彩令牌

> 全部界面颜色**只能**来自本节令牌（CSS 变量前缀 `--ip-`）。硬编码色值由门禁拦截。

## 3. 字体与字阶

- 字族：`MiSans, HarmonyOS Sans SC, PingFang SC, Microsoft YaHei, sans-serif`；数字场景加 `font-variant-numeric: tabular-nums`。
- **字阶 8 档（禁用其他字号）**：

| 令牌 | 值/行高 | 用途 |
|---|---|---|
| `--ip-font-caption` | 12px / 1.5 | 角标、表格辅助 |
| `--ip-font-hint` | 13px / 1.5 | 提示、时间戳 |
| `--ip-font-body` | 14px / 1.6 | 正文、表单、表格 |
| `--ip-font-emphasis` | 16px / 1.6 | 强调正文、列表标题 |
| `--ip-font-title-sm` | 20px / 1.4 | 卡片标题、弹窗标题 |
| `--ip-font-title` | 24px / 1.35 | 区块标题 |
| `--ip-font-headline` | 30px / 1.25 | 页面标题 |
| `--ip-font-display` | 38px / 1.2 | 门户首屏品牌标题（唯一允许处） |

- 字重仅 `400 / 500 / 600 / 700`。**禁止 800 及以上**（旧规范中的 800–850 全部降为 700）。
- 中文标题禁止出现孤字断行；两行标题使用 `text-wrap: balance` 或手工断句。

## 4. 间距 · 圆角 · 阴影 · 动效

- **间距 8 档**：`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`（px）。禁止出现此外的 margin/padding 值。
- **圆角 4 档**：`--ip-radius-sm: 6px`（输入、按钮、标签）/ `--ip-radius-md: 10px`（卡片、弹窗）/ `--ip-radius-lg: 16px`（门户大卡）/ `--ip-radius-full: 999px`（胶囊、头像）。
- **阴影 3 档（elevation）**，默认无阴影，需要浮起才用：
  - `--ip-shadow-sm`: `0 1px 2px rgba(11,24,51,.06)` — 卡片静置
  - `--ip-shadow-md`: `0 4px 12px rgba(11,24,51,.08)` — 悬停、下拉
  - `--ip-shadow-lg`: `0 12px 32px rgba(11,24,51,.12)` — 弹窗、抽屉
- **动效**：时长 `120ms`（微反馈）/ `200ms`（常规）/ `320ms`（面板进出）；缓动统一 `cubic-bezier(0.2, 0, 0, 1)`。禁止无目的动画（如秒级时钟跳动这类持续性噪音）。

## 5. 布局

- 基准画布 1440px；内容区最大宽 `--ip-layout-max: 1440px`，居中；**任何断点下禁止横向裁切内容**（栅格列数随容器收敛）。
- 断点：`<768 单列 / 768–1199 双列 / ≥1200 三列`（卡片栅格用 `repeat(auto-fill, minmax(320px, 1fr))` 类策略，不写死列数）。
- 门户模块页统一骨架：左侧品牌导航栏（260px）+ 主区（工具条 + 内容面板）。骨架结构与间距全站一致，模块间只允许 accent 与图标不同。
- 页面纵向留白：内容结束后页脚上限留白 64px，禁止整屏空置（数据少时由空态组件填充，§8.4）。

## 6. 图标

- **唯一来源：Iconify**（项目内以 `~icons/*` 引入），线性风格、1.75px 视觉笔画、圆头。
- 尺寸档：16 / 20 / 24 / 32。颜色只用 neutral-500 / primary / 语义色 / 模块色。
- **禁止**：AI 生成的 3D/立体/渐变插画混作功能图标；同一区域混用面性与线性；随机彩色底字母块（应用 LOGO 无图时用模块 soft 底 + base 色首字母，全站同一规则）。
- 门户首页五张模块插画属于品牌资产：必须成套委托/生成（同一风格、同一光源、同一色系约束——限定品牌蓝 + 对应模块色），任何单张替换都需整套评审。

## 7. 组件规约

### 7.1 按钮
- 层级四档：**primary**（实心品牌蓝，**每屏至多一个**）→ **secondary**（描边）→ **tertiary**（文字/幽灵）→ **danger**（实心 danger，仅破坏性操作）。
- 高度：36px（默认）/ 32px（表格内）/ 44px（移动端）；圆角 `--ip-radius-sm`。
- 同一容器内多按钮：主次分明，禁止两个实心深色并排（现应用中心页"搜索+需求反馈"即反例）。

### 7.2 卡片
- 圆角 `--ip-radius-md`，边框 neutral-200，阴影至多 `sm`（悬停可升 `md`）。
- 栅格内卡片**等高**（flex/grid 拉伸），操作区吸底对齐；标题至多两行省略，禁止孤字断行。
- 整卡可点时不再放"进入"按钮（首页卡"整卡+圆形箭头钮"二选一，保留整卡点击 + 右下角箭头视觉指示即可）。

### 7.3 表单 / 表格 / 标签
- 输入高 36px、圆角 sm、聚焦 primary 边框 + 2px primary-100 外圈。
- 表格：表头 neutral-100 底 + neutral-600 字；时间列固定 `YYYY-MM-DD HH:mm` 宽度，禁止截断出省略号；状态一律用语义 soft 标签。
- 标签（tag）：soft 底 + base 字 + border 描边，高 22px，圆角 sm。

### 7.4 五态（每个数据界面必须交付）
- **加载**：骨架屏（卡片/表格两种通用骨架），禁止白屏或仅转圈超过 300ms。
- **空数据**：插图（线性风格）+ 一句引导 + 主行动按钮（如"暂无话题，发起第一个讨论"）。
- **错误**：说明 + 重试按钮；禁止裸 toast 后留白页。
- **无权限**：统一 403 视图。
- **零值指标**：数据看板类零值显示 `—` 或隐藏该卡，禁止渲染"0 次"矩阵；只有一个分类时禁止使用饼/环图（改用数字大字卡）。

## 8. 内容与文案

- 术语以 BC 术语表为准：模块名全站唯一（"资料共享"不得写作"资源共享"；AppCenter 面向用户统一写作"应用中心"）。
- 数字：千分位分隔；单位与数字间留 1 空格；时间统一 `YYYY-MM-DD HH:mm`（秒仅日志场景）。
- 演示/占位数据不得出现在生产界面（默认昵称、测试菜单、示例单位等入库前清理）。
- 语气：简体中文、动宾短语按钮（"上传资料"），禁止中英混排装饰词（如板块标题下的英文小字仅品牌区可用）。

## 9. 可访问性（硬性）

- 正文对比度 ≥ 4.5:1，大字号 ≥ 3:1（按令牌组合预校验，见 tokens.scss 注释）。
- **所有可交互元素必须有可见焦点态**：`:focus-visible` 显示 2px primary 外圈（令牌 `--ip-focus-ring`），全站统一，由基础样式层提供，禁止 `outline: none` 裸移除。
- 点击目标：桌面 ≥ 36px，移动 ≥ 44px。
- 图片/图标按钮必须有 aria-label；颜色不得是传达状态的唯一手段（配文字或图标）。

## 10. 门禁与流程（本规范的执行机制）

1. **设计审计门禁**：`npm run design:audit`（`deploy/scripts/design-audit.mjs`）统计门户代码中硬编码色值/字号/圆角的离散度，与 `plus-ui/design-audit-baseline.json` 棘轮基线比较——**超过基线即构建失败**；随迁移进度只允许下调基线，禁止上调。目标终态：硬编码色值归零，全部走令牌。
2. **截图回归**：`deploy/scripts/ui-capture.py` 采集关键页面截图（登录/门户四页/后台三页/移动端），重大样式改动前后各跑一次并对比归档。
3. **评审清单**（合并前自查）：只用令牌？五态齐全？每屏一个 primary？无孤字断行/截断？焦点态可见？术语与 BC 一致？
4. 本规范与 `tokens.scss` 同步修订；任何"规范说 A 代码是 B"都按缺陷处理。
