# 信息中心数智服务平台 UI/UX 设计规范

## 1. 规范来源

本平台采用“系统级设计原则 + 企业 Web 落地规则”的组合，不直接照搬某一家系统。

- Apple Human Interface Guidelines：https://developer.apple.com/design/human-interface-guidelines/
- HarmonyOS Design：https://developer.huawei.com/consumer/cn/design/
- Ant Design：https://ant.design/docs/spec/introduce-cn
- Element Plus Design：https://element-plus.org/en-US/guide/design.html

以上来源分别用于参考清晰层级、可读性、交互反馈、企业 Web 布局、数据密集界面和当前组件库行为一致性。

## 2. 设计目标

- 稳重可信：适合信息中心、内部服务、政务/企事业场景。
- 高效可扫：用户能快速找到资料、工具、论坛等高频入口。
- 比例克制：界面像专业系统，不做海报化、过大化、装饰化。
- 一致可复用：间距、字号、圆角、按钮、卡片、图标风格统一。

## 3. 画布与布局

- 设计基准画布：1440px 宽。
- 重点适配宽度：1280、1366、1440、1680、1920。
- 页面内容区应设置最大宽度，不随超宽屏无限拉伸。
- 采用 8px 栅格系统，常用间距为 8、16、24、32、48、64。
- 避免 29、37、188 这类无规则尺寸，除非来自明确视觉资产裁切。

## 4. 字体层级

- 正文：14px 或 16px。
- 表单标签、辅助说明：13px 或 14px。
- 卡片标题：20px 到 28px。
- 页面标题：28px 到 36px。
- 门户首屏品牌标题：32px 到 40px，不再继续放大。
- 行高默认 1.35 到 1.6，按钮和数字类元素可收紧到 1。
- 字重使用 400、500、600、700、800，避免大面积 900 以上字重。

## 5. 控件尺寸

- 桌面端最小点击目标：36px。
- 移动端最小点击目标：44px。
- 普通按钮高度：36px、40px、44px。
- 大型门户入口按钮可使用 48px，但不应影响卡片比例。
- 圆形图标按钮必须保持宽高一致，并设置 `flex: 0 0 size` 防止被压扁。

## 6. 卡片规则

- 普通业务卡片圆角：8px 到 12px。
- 门户首屏大入口卡片圆角：18px 到 24px。
- 卡片内图标不超过卡片高度的 48%。
- 大入口卡片宽高比宜控制在 0.75 到 0.9 之间，避免过瘦或过扁。
- 卡片阴影应轻，不使用重投影制造廉价浮层感。

## 7. 图标规则

- 优先使用同一来源、同一视觉语言的图标资产。
- 可选来源：Element Plus Icons、Iconify、Iconfont 下载图标、统一设计资产。
- 不使用临时手绘 SVG 作为正式图标。
- 同一区域图标应统一线性、面性、渐变、透视和阴影风格。
- 图标只服务识别，不承担大面积装饰功能。

## 8. 响应式规则

- 1366px 及以上：门户入口一行 5 个。
- 1024px 到 1365px：仍优先一行 5 个，缩小卡片和图标。
- 768px 到 1023px：入口改为 3 列，避免 2 列巨型卡片。
- 640px 到 767px：入口改为 2 列。
- 640px 以下：入口单列，统计项单列或双列按内容决定。
- 顶部品牌区和状态区在 900px 以上尽量保持同一行，减少首屏垂直浪费。

## 9. 门户首页比例基准

- 首屏主体应完整露出品牌区、模块入口和统计概览的上半部分。
- 背景主视觉不能被卡片完全挡住，应保留核心景观识别。
- 模块卡片高度桌面端控制在 320px 到 350px。
- 模块图标桌面端控制在 150px 到 176px。
- 模块标题控制在 24px 到 28px。
- 统计条高度控制在 96px 到 112px，移动/窄屏时允许换行堆叠。

## 10. 执行要求

- 新页面先按本规范定义尺寸，再进入开发。
- 页面完成后至少验证一个桌面宽度和一个窄屏宽度。
- 视觉验收必须看真实浏览器截图，不只看代码。
- 若页面使用新增图标资产，应记录来源并保证同风格。
