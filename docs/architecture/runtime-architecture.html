info-serve 当前运行时架构

Docker Compose · 门户五服务 · 同步边界与 portal.topic 异步协作

当前入口与服务

  • • 用户只经 nginx-web :7010 与 Gateway :8180 进入门户
  • • Gateway 依据 Nacos 路由到 Kernel、AppCenter、Forum、RequiredKnowledge、Resources

同步调用边界

  • • 普通实线只表示代码或配置已确认的当前同步依赖
  • • Kernel → AppCenter / Forum / Resources 仅用于统计，2 秒超时，异常或空结果降级为 0
  • • 业务服务允许调用通用服务；未确认的允许方向不画成实际连线

异步协作

  • • 跨 BC 业务协作统一经 RabbitMQ portal.topic
  • • AppCenter、Forum、Resources 发布通知事件；Kernel 消费通知
  • • Resources 还使用 portal-resources.convert 与 DLQ 处理文档转换

禁止方向

  • • 禁止业务 BC ↔ BC 直接同步互调
  • • 禁止跨域 SQL；禁止复制 Kernel 的消息、收藏、统计与注册表
  • • 禁止绕过 Gateway 暴露门户业务入口

数据与配置

  • • 所有 Java 服务使用 Nacos；Nacos 的 MySQL 仅保存 ry-config
  • • 业务共享 PostgreSQL 实例，但按 app_ / forum_ / rk_ / res_ / portal_ 表前缀逻辑隔离
  • • Redis 承担缓存、令牌、转换去重与通知事件幂等

文件数据面

  • • AppCenter、Resources 经 RemoteFileService 获取文件契约或元数据
  • • File、AppCenter、Resources 均可经 OssFactory 访问 MinIO 对象数据面
